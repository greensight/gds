# Colors

## Палитры проекта

✨ Theme: `palettes`

✨ Tokens: `palettes`

✨ Autokit: `Palettes`

Из этих значений дизайнер строит конечные переменные в `Colors`. Разработчик непосредственно значения из `Palettes` не использует. Эти значения тянутся для демонстрации паттернов на уровне самой дизайн-системы. На уровне проекта их можно вывести через автокит, но этим их использование ограничивается.

**Интерфейс:**

```js
interface IPalettes {
    /** Набор цветов палитры */
    [name: string]: string[];
}
```

## Цвета проекта

✨ Theme: `colors`

✨ Tokens: `colors`

✨ Autokit: `Colors`

Это финальные переменные, на которые заводятся стили. Вы должны для всех цветов проекта использовать только эти переменные.

⛔️ Если вы используете токены, то не должны сами добавлять переменные. Если появилась такая необходимость, то вероятнее всего, что это ошибка макета. Обсуждайте такие моменты с ведущим дизайнером проекта.

**Интерфейс:**

```js
interface IColors {
    /** Переменная цвета */
    [name: string]: string;
}
```

## Использование

Нужно задать цвет фона для некоторого элемента. В макете у этого элемента указан стиль:

<img src="images/colors/style.png" alt="Стиль цвета в Figma" width="300" />

_Стиль цвета в Figma_

Вы знаете имя стиля и можете обратиться к соответствующей переменной темы:

```js
import { useTheme } from '@greensight/gds';

const Example = () => {
    const { colors } = useTheme();
    return <div css={{ backgroundColor: colors.brand }}>Example</div>;
};
```

⚠️ Дизайнеры могут группировать стили для собственного удобства. Это выражается слешами в названиях - их просто игнорируйте.
