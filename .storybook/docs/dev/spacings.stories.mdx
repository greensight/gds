# Spacings

✨ Utils: `scale`

Для задания размеров элементов, полей и отступов вместо прямого указания числа пикселей используется хелпер `scale`. По умолчанию он работает по major scale, при передаче вторым параметром `true` включается minor scale.

Пример:

```js
<Example
    css={{
        width: scale(20),
        height: scale(6),
        padding: `${scale(1, true)}px ${scale(1)}px`,
        marginBottom: scale(2),
    }}
/>
```

## Изменение шага сетки

Если на проекте базовый размер сетки отличается от 8, то вам нужно переопределить хелпер `scale` под ваш проект. Для этого нужно переопределить хелпер с вызовом 3-го параметра и вызывать его вместо прямого вызова хелпера из пакета GDS.

В отдельном файле:

```js
import { scale as gdsScale } from '@greensight/gds';

const scale = (multiplier, isMinor) => gdsScale(multiplier, isMinor, 10);

export default scale;
```

⛔️ Это единственное место на проекте, где должен быть указан 3-ий параметр. Не допускается переопределение шага по месту, хотя формально система это позволяет сделать. Можете предложить pull request с более элегантным подходом к конфигурации хелпера.

## Общие правила

1. Использовать хелпер `scale` для задания всех размеров, где имеет место сетка: padding - хорошо, borderRadius - пишем в px.
1. minor не определён для размеров выше 40px. Хелпер выведет предупреждение и округлит результат до 8.
1. В качестве множителя принимаются только целые значения. Хелпер выведет предупреждение и округлит множитель до целого.
1. Применять minor scale только для размеров, которые невозможно представить через major (некратные 8): `scale(2)` - хорошо, `scale(4, true)` - бессмысленно.
