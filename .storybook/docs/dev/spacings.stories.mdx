# Spacings

Для задания размеров элементов, полей и отступов используются шкалы major и minor - по 8 и по 4 px соответственно. Для их использования прописан хелпер `scale`. По умолчанию он работает по major scale. minor scale включается при передаче вторым параметром `true`.

## Пример

```js
<SomeComponent
    css={{
        width: scale(20),
        height: scale(6),
        padding: `${scale(1, true)}px ${scale(1)}px`,
        marginBottom: scale(2),
    }}
/>
```

## Изменение шага сетки

Для использования шага, отличного от 8, нужно переопределить хелпер `scale` на проекте с передачей 3-го параметра и вызывать его вместо прямого вызова хелпера из пакета GDS.

В отдельном файле:

```js
import { scale as gdsScale } from '@greensight/gds';

const scale = (multiplier, isMinor) => gdsScale(multiplier, isMinor, 10);

export default scale;
```

## Общие правила

1. Использовать хелпер `scale` для задания всех размеров, где имеет место сетка: padding - хорошо, borderRadius - пишем в px.
1. minor не определён для размеров выше 40px. Хелпер выведет предупреждение и округлит результат до 8.
1. В качестве множителя принимаются только целые значения. Хелпер выведет предупреждение и округлит множитель до целого.
1. Применять minor scale только для размеров, которые невозможно представить через major (некратные 8): `scale(2)` - хорошо, `scale(4, true)` - бессмысленно.
1. Не использовать кастомный шаг сетки нигде кроме единственного скрипта с переопределением хелпера. Весь проект должен работать на едином шаге.
