# Typography

✨ Tokens: `Typography`

Фрейм `Typography` Figma хранит данные о типографике проекта. Они могут быть синхронизированы с разработкой через дизайн-токены. Решение диктует строгость в использовании стилей, экономит время разработки и упрощает корректировку значений.

<img src="images/typography/frame.png" alt="Фрейм Typography" width="600" />

_Фрейм Typography_

Дизайнер подбирает типографику и меняет параметры по предложенному шаблону. Новые переменные добавляются в конец списка, соблюдая оригинальную вложенность и именование слоев. Из подобранных значений дизайнер создает текстовые стили Figma и использует их на проекте.

## Структура фрейма

Синим цветом обозначены слои, в которые дизайнер вносит изменения.

<img src="images/typography/structure.png" alt="Структура фрейма Typography" width="600" />

_Структура фрейма Typography_

`description` - вспомогательный фрейм для подписей, он не экспортируется.

`breakpoints` задаёт точки, которым соответствуют стили из фреймов `desktop` и `mobile`.

Названия переменных во фреймах `desktop` и `mobile` должны совпадать. Допускается, что во фрейме `mobile` отсутствуют переменные,
которые есть в `desktop` - это делает размер фиксированным.

## Флюидная типографика

По умолчанию для текста на проектах применяется плавное изменение размера кегля между двумя определёнными брейкпоинтами. Вне этого диапазона размер фиксируется, чтобы избежать чрезмерного уменьшения/увеличения.

Брейкпоинты указываются во фрейме `breakpoints`. Для GDS это **1440** и **768**.

⛔️ Число брейкпоинтов более 2 не поддерживается.

**При разрешении экрана 1920** пользователю отобразятся размеры из набора фрейма `desktop`.

**При разрешении экрана 375** пользователю отобразятся размеры из набора фрейма `mobile`.

**При разрешении экрана 960 или 1280** пользователю отобразятся автоматически подобранные значения в интервале между **768** и **1440**.

⚠️ Если вам по какой-то причине не нужно плавное изменение шрифта, то его можно отключить на разработке либо полностью, либо для отдельных стилей. Уведомите об этом разработчика.

## Нейминг

<img src="images/typography/variables.png" alt="Наименования переменных" width="450" />

_Наименования переменных_

Для проекта нам требуется минимальный набор типографических переменных:

-   **h1, h2, h3, h4, h5, h6** — стили заголовков
-   **subheading** — подзаголовки
-   **body** — основной текст
-   **small** — мелкие тексты
-   **caption** — подписи
-   **button** — тексты на кнопках

Задача дизайнера не только определить стили типографики, но и присвоить им переменные.

Правила:

- Переменные используют формат [lowerCamelCase](https://ru.wikipedia.org/wiki/CamelCase): первое слово переменной начинается со строчной, каждое последующее — с прописной. Несколько слов пишутся слитно без пробела, дефиса или нижнего подчеркивания.
- Мы не используем транслитерацию — все слова пишутся на английском.

## Рекомендации

1. Указывайте line-height в процентах. Это позволит одинаково выставлять значения для разных кеглей и не менять значение при смене кегля. Разумно использовать значение, кратное десяти: 120%, 130%, 140%, 150% и т.д.
1. Не используйте более двух разных нестандартных шрифтов в проекте, если это ничем не оправдано. Скорость загрузки страницы увеличивается пропорционально количеству начертаний.
1. Предоставляйте разработчикам вариативную версию шрифта, если она имеется. На текущий момент поддержки вариативных шрифтов в Figma нет, но его можно вставить вместо обычной версии на этапе разработки, что значительно понизит общий вес загружаемых ассетов.
1. Подбирайте фоллбеки для нестандартных шрифтов. Фоллбек — системный шрифт, который видит пользователь, пока не загрузится нестандартный шрифт. Для уменьшения визуального скачка подбирайте похожий по параметрам шрифт из системных.
1. Не используйте line-height в качестве отступов при построении вертикальных списков. Это рушит логику работы с текстом, а при переносе строки мы имеем неадекватный line-height. Это мешает разработчикам выставить точные показатели. Это нарушает работу со стилями.

## Ресурсы для построения ритма кегля

Мы рекомендуем инструменты, которые помогут дизайнеру спроектировать равномерный ритм размера шрифта в проекте. Сервисы — не панацея, а инструмент помощи и системный подход к построению типографики.

-   [Modular Scale](https://www.modularscale.com/)
-   [A Visual Type Scale](https://type-scale.com/)
