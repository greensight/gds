# Icons

Фрейм `Icons` отвечает за автоматический экспорт иконок из Фигмы в Сторибук. Иконки экспортируются в квадратах указанных размеров.

Каждая иконка состоит из векторных слоев и обертки-компонента.

<img src={iconsStructure} style={{ display: 'block', margin: '0 auto' }} />

Тесты пока не выявили проблем с использованием оутлайновых иконок (параметр stroke). Все же, готовить иконки для экспорта предпочтительнее с использованием параметра fill.

В Сторибук экспортируются только компоненты, другие типы слоев не затрагиваются.

## Проектирование и размеры иконок в GDS

Иконки рисуются на базе 24-х пиксельного квадрата, имеют форму прямоугольника или круга:

<img src={iconsForm} style={{ display: 'block', margin: '0 auto' }} />

Строго соблюдать сетку выше не следует, нужно больше ориентироваться на визуальную оценку. Сетка лишь дает подсказку.

Чтобы толщины линий в иконках привязывались к пиксельной сетке (в случае заливки _fill_), мастер-иконку нужно рисовать с толщиной линии 2 px в квадрате 24 px. Тогда толщина линий при скэйлинге составит:

<img src={iconsSize} style={{ display: 'block', margin: '0 auto' }} />

Построение иконок и предложенная размерность может отличаться, если этого требует проект. Это рекомендация, а не панацея.

## Привязки

Интерфейсные иконки используются в элементах, поэтому желательно сразу настроить привзяки.

-   Для масштабируемых иконок — scale / scale
-   Для иконок фиксированных размеров —  center / center

## Нейминг

Кроме отрисовки иконок в задачу дизайнера входит их структурирование. На основе аналитики мы выделили структуру слоев и назначили нейминг:

<img src={iconsNames} style={{ display: 'block', margin: '0 auto' }} />

`size` — группа размеров, в которую входит иконка. Допустимо использовать цифру из стороны квадрата (24) или иные обозначения размеров (middle, small).

`name` — уникальное название иконки, по которой ее можно будет легко найти.

`type` — тип иконки. Иконка может быть оутлайновой (stroke) или с заливкой (fill). Тип может отражать направление стрелки или другую характеристику.

При указании имен используется формат lowerCamelCase.

Иконка должна заключаться в строгий квадрат — это гарантирует правильное отображение иконки в макетах и после разработки.

## Компонентный подход

Чтобы иконки масштабировались и их было удобно использовать в макетах, каждый размер сделан отдельным компонентом, который выполняет роль обертки. Такой подход экономит время в случае корректировки мастер-иконки.

<img src={iconsComponents} style={{ display: 'block', margin: '0 auto' }} />

Рекомендация: мастер-иконка рисуется один раз и дублируется для изменения размера. Новая иконка оборачивается в компонент и именуется в соответствии со структурой. Такой подход позволяет быстрее корректировать иконки.

## Стандартная коллекция

В СК входят иконки, которые повторяются в каждом проекте и выглядят одинаково.

<img src={iconsStandart} style={{ display: 'block', margin: '0 auto' }} />

Для упрощения дальнейшей стилизации иконки отрисованы оутлайнами в размере 24×24 px. Стандартная коллекция по умолчанию экспортируется в Сторибук, однако дизайнер может масштабировать или изменить иконки по своему усмотрению.

Приветствуется пополнение стандартной коллекции ГДС для ускорения процесса работы над макетами.

## Рекомендации

1. Хорошо когда в ките одинаковая логика иконок: стиль, толщина линий, цвета, размеры, расположение, скругление углов.
2. Нужно занести в кит максимальное кол-во используемых иконок. Это ускорит процесс разработки.
3. Иконки, выполняющие роль иллюстраций или подчеркивающие стиль могут не подчиняться правилам и не заноситься в кит.
4. Одноцветные иконки состоят из одного слоя, если это возможно.
5. Границы объектов привязываются к пиксельной сетке.
