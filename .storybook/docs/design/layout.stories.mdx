# Layout

✨ Tokens: `Layout`

Фрейм `Layout` определяет основную сетку и брейкпоинты проекта.

<img src="images/layout/frame.png" alt="Фрейм Layout" />

_Фрейм Layout_

Проект содержит 9 брейкпоинтов, на которых возможны изменения в сетке. Кол-во брейкпоинтов не означает необходимость отрисовки макетов под каждый - они определяют набор точек, на которых разработчик может осуществить переход.

На каждом брейкпоинте определяется:

- значение брейкпоинта
- число колонок
- отступы между колонками
- ширина контейнера
- выравнивание контейнера
- поля

⛔️ Количество брейкпоинтов менять нельзя, т.к. дизайнер не задаёт имена переменных, и они зашиваются со стороны системы в расчёте на то, что их будет ровно 9.

При токенизации значения берутся из фактического размера прямоугольников и фреймов, так что дизайнер следит, чтобы границы фреймов соответствовали значению брейкпоинта.

<img src="images/layout/breakpoint-good.png" alt="Правильно: размер фрейма соответствует брейкпоинту" />

_Правильно: размер фрейма соответствует брейкпоинту_

<img src="images/layout/breakpoint-bad.png" alt="Неправильно: размер фрейма больше брейкпоинта" />

_Неправильно: размер фрейма больше брейкпоинта_

## Использование контейнера

Самый большой брейкпоинт определяет его поведение на больших разрешениях. На примере GDS, контейнер фиксируется по ширине и выравнивается по центру.

Наличие контейнера определяется по значению `auto` у элемента `padding`. Контейнер можно центрировать или выравнять по левому краю. Без указания `auto` макет становится полностью резиновый.
